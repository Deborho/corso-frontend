2. Input Groups

Gli input group permettono di aggiungere testo, pulsanti o altri elementi prima o dopo un input. Utili per creare interfacce più chiare e funzionali.

2.1 Input Group Base

<!-- Testo prima -->
<div class="input-group mb-3">
    <span class="input-group-text">@</span>
    <input type="text" class="form-control" placeholder="Username">
</div>

<!-- Testo dopo -->
<div class="input-group mb-3">
    <input type="text" class="form-control" placeholder="Recipient's username">
    <span class="input-group-text">@example.com</span>
</div>

<!-- Testo prima e dopo -->
<div class="input-group mb-3">
    <span class="input-group-text">$</span>
    <input type="text" class="form-control">
    <span class="input-group-text">.00</span>
</div>

2.2 Input Group con Button

5. Validation

Bootstrap fornisce classi per visualizzare feedback di validazione. Puoi usare la validazione HTML5 nativa o custom con JavaScript.

5.1 Validation States

<form>
    <!-- Valid feedback -->
    <div class="mb-3">
        <label for="validInput" class="form-label">Valid input</label>
        <input type="text" class="form-control is-valid" id="validInput" 
               value="Valid value">
        <div class="valid-feedback">
            Looks good!
        </div>
    </div>
    
    <!-- Invalid feedback -->
    <div class="mb-3">
        <label for="invalidInput" class="form-label">Invalid input</label>
        <input type="text" class="form-control is-invalid" id="invalidInput">
        <div class="invalid-feedback">
            Please provide a valid value.
        </div>
    </div>
</form>

<script>
// JavaScript per abilitare la validazione Bootstrap
(function () {
    'use strict'
    var forms = document.querySelectorAll('.needs-validation')
    Array.prototype.slice.call(forms).forEach(function (form) {
        form.addEventListener('submit', function (event) {
            if (!form.checkValidity()) {
                event.preventDefault()
                event.stopPropagation()
            }
            form.classList.add('was-validated')
        }, false)
    })
})()
</script>

8. Best Practices

✓ Consigli:

  • Usa sempre <label> associato a ogni input (con for)
  • Aggiungi .mb-3 ai wrapper per spacing consistente
  • Usa placeholder solo per esempi, non per label
  • Implementa validazione sia lato client che server
  • Fornisci feedback chiaro per errori di validazione
  • Usa input group per rendere gli input più chiari
  • Floating labels per form moderne e pulite

✗ Evita:

  • Non usare solo placeholder senza label (accessibilità)
  • Non dimenticare attributi name sui form controls
  • Non usare disabled quando serve readonly
  • Non omettere feedback di validazione
  • Non fare form troppo lunghi (dividi in step)
