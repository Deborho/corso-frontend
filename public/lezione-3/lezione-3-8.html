Come personalizzare Bootstrap per i tuoi progetti

1. CSS Custom Properties (Variabili CSS)

Bootstrap 5 usa CSS custom properties (variabili CSS) che possono essere sovrascritte facilmente per personalizzare colori, spacing e altri valori.

1.1 Variabili Principali

:root {
    /* Colori primari */
    --bs-primary: #0d6efd;
    --bs-secondary: #6c757d;
    --bs-success: #198754;
    --bs-info: #0dcaf0;
    --bs-warning: #ffc107;
    --bs-danger: #dc3545;
    --bs-light: #f8f9fa;
    --bs-dark: #212529;
    
    /* Font */
    --bs-body-font-family: system-ui, -apple-system, "Segoe UI", Roboto;
    --bs-body-font-size: 1rem;
    --bs-body-font-weight: 400;
    --bs-body-line-height: 1.5;
    
    /* Spacing */
    --bs-gutter-x: 1.5rem;
    --bs-gutter-y: 0;
    
    /* Border */
    --bs-border-width: 1px;
    --bs-border-radius: 0.375rem;
    --bs-border-radius-lg: 0.5rem;
    --bs-border-radius-sm: 0.25rem;
}

3. Creare un Tema Custom

Puoi creare un tema completo modificando le variabili CSS e aggiungendo stili custom. Ecco un esempio di tema dark.

3.1 Dark Theme

<style>

[data-bs-theme="dark"] {
    /* Colori base */
    --bs-body-color: #e4e4e7;
    --bs-body-bg: #18181b;
    
    /* Colori testo */
    --bs-heading-color: #fafafa;
    --bs-link-color: #a78bfa;
    --bs-link-hover-color: #c4b5fd;
    
    /* Borders */
    --bs-border-color: #27272a;
    
    /* Componenti */
    --bs-primary: #a78bfa;
    --bs-secondary: #71717a;
    
    /* Card */
    --bs-card-bg: #27272a;
    --bs-card-border-color: #3f3f46;
    
    /* Input */
    --bs-body-bg: #27272a;
    --bs-form-control-bg: #18181b;
    --bs-form-control-color: #e4e4e7;

/* Applica dark theme */

    background-color: var(--bs-body-bg);

<script>
const themeToggle = document.getElementById('themeToggle')

// Carica tema salvato

    htmlElement.setAttribute('data-bs-theme', newTheme)
    localStorage.setItem('theme', newTheme)
    
    // Aggiorna icona button
    themeToggle.innerHTML = newTheme === 'dark'

4. Utilities API

Bootstrap 5 include una Utilities API che permette di creare utility classes custom senza scrivere CSS manualmente.

4.1 Creare Utilities Custom (con SASS)

// custom.scss

// Import Bootstrap functions e variables
@import "bootstrap/scss/functions";
@import "bootstrap/scss/variables";
@import "bootstrap/scss/maps";
@import "bootstrap/scss/mixins";
@import "bootstrap/scss/utilities";

// Aggiungi utilities custom
$utilities: map-merge(
    $utilities,
    (
        // Font size custom
        "font-size": (
            property: font-size,
            class: fs,
            values: (
                1: 3rem,
                2: 2.5rem,
                3: 2rem,
                4: 1.75rem,
                5: 1.5rem,
                6: 1.25rem,
                7: 1rem,
                8: 0.875rem
            )
        ),
        // Opacity custom
        "opacity": (
            property: opacity,
            class: opacity,
            values: (
                0: 0,
                10: .1,
                20: .2,
                30: .3,
                40: .4,
                50: .5,
                60: .6,
                70: .7,
                80: .8,
                90: .9,
                100: 1
            )
        ),
        // Letter spacing
        "letter-spacing": (
            property: letter-spacing,
            class: tracking,
            values: (
                tighter: -0.05em,
                tight: -0.025em,
                normal: 0,
                wide: 0.025em,
                wider: 0.05em,
                widest: 0.1em
            )
        )
    )
);

// Import Bootstrap
@import "bootstrap/scss/bootstrap";

4.2 Usare Utilities Custom

<!-- Dopo aver compilato il SASS custom, puoi usare: -->

<h1 class="fs-1 tracking-wide">Big Title</h1>
<p class="fs-7 opacity-70">Small text with opacity</p>

7. Best Practices per Customizzazione

‚úì Consigli:

  • Usa variabili CSS per modifiche globali e consistenti
  • Carica sempre il tuo CSS custom dopo Bootstrap
  • Documenta tutte le personalizzazioni per manutenibilit√†
  • Testa le modifiche su tutti i componenti Bootstrap
  • Mantieni la compatibilit√† con breakpoints responsive
  • Crea classi utility riusabili invece di stili inline
  • Usa CSS specificity correttamente per evitare !important
  • Considera l'accessibilit√† nei colori custom (contrasto)

‚úó Evita:

  • Non modificare direttamente bootstrap.min.css
  • Non abusare di !important
  • Non sovrascrivere troppe classi (meglio estendere)
  • Non ignorare il responsive design nelle personalizzazioni
