Lezione 4.1 - Introduzione e Setup SASS

Cos'è SASS, perché usarlo e come configurarlo

1. Cos'è SASS/SCSS?

SASS (Syntactically Awesome Style Sheets) è un preprocessore CSS, ovvero un linguaggio che estende CSS aggiungendo funzionalità di programmazione come variabili, nesting, mixins, functions e molto altro.

Il codice SASS viene scritto in file con estensione .sass o .scss, poi viene compilato in CSS standard che il browser può interpretare.

Vantaggi di SASS:

  • Variabili: Memorizza valori riutilizzabili (colori, font, spacing)
  • Nesting: Struttura gerarchica che riflette l'HTML
  • Mixins: Blocchi di codice riutilizzabili con parametri
  • Functions: Calcoli e trasformazioni dinamiche
  • Partials: Divide il CSS in file modulari
  • Ereditarietà: @extend per condividere stili tra selettori
  • Operatori: Math operations e string manipulation
  • Control Directives: Logica condizionale e loops

2. SASS vs SCSS: Le Due Sintassi

SASS (Indented Syntax) SCSS (Sassy CSS) 
// File: style.sass
$primary-color: #3498db

.button
  background: $primary-color
  padding: 10px 20px
  border: none
  &:hover
    background: darken($primary-color, 10%)
 
// File: style.scss
$primary-color: #3498db;

.button {
  background: $primary-color;
  padding: 10px 20px;
  border: none;
  &:hover {
    background: darken($primary-color, 10%);
  }
}
  • Basata su indentazione (come Python)
  • Niente graffe {} o punto e virgola ;
  • Più concisa ma meno flessibile
  • Estensione .sass
  • Sintassi identica a CSS
  • Usa graffe {} e punto e virgola ;
  • Più familiare per chi conosce CSS
  • Estensione .scss
  • PIÙ USATA E RACCOMANDATA

In questo corso useremo SCSS perché:

  • È retrocompatibile con CSS (ogni CSS valido è SCSS valido)
  • È più diffusa nell'industria
  • È più facile convertire CSS esistente in SCSS
  • Supportata da tutti i tool e framework moderni

3. Perché Usare SASS?

3.1 Problema: CSS Tradizionale

/* CSS tradizionale - ripetitivo e difficile da mantenere */
.header {
  background: #3498db;
  color: white;
  padding: 20px;
}

.header .nav {
  display: flex;
}

.header .nav .nav-item {
  padding: 10px;
}

.header .nav .nav-item:hover {
  background: #2980b9; /* devo ricordare quale colore più scuro usare */
}

.button-primary {
  background: #3498db; /* stesso colore ripetuto! */
  padding: 10px 20px;
}

.button-secondary {
  background: #2ecc71; /* un altro colore da ricordare */
  padding: 10px 20px; /* padding ripetuto! */
}

3.2 Soluzione: SASS/SCSS

// SCSS - organizzato, riutilizzabile, mantenibile
$primary-color: #3498db;
$secondary-color: #2ecc71;
$padding-base: 10px 20px;

@mixin button-base {
  padding: $padding-base;
  border: none;
  cursor: pointer;
}

.header {
  background: $primary-color;
  color: white;
  padding: 20px;
  
  .nav {
    display: flex;
    
    .nav-item {
      padding: 10px;
      
      &:hover {
        background: darken($primary-color, 10%); // calcolo automatico!
      }
    }
  }
}

.button-primary {
  @include button-base; // riuso il codice!
  background: $primary-color;
}

.button-secondary {
  @include button-base;
  background: $secondary-color;
}

4. Installazione SASS

4.1 Prerequisito: Node.js e npm

SASS richiede Node.js per l'installazione. Verifica se hai già Node.js installato:

# Verifica installazione Node.js
node --version
# Output esempio: v18.17.0

# Verifica installazione npm (incluso con Node.js)
npm --version
# Output esempio: 9.6.7

Se non hai Node.js, scaricalo da nodejs.org (versione LTS consigliata).

4.2 Installazione SASS

Ci sono due modi per installare SASS:

Opzione 1: Installazione Globale (consigliata per iniziare)

# Installa SASS globalmente
npm install -g sass

# Verifica installazione
sass --version
# Output esempio: 1.69.5 compiled with dart2js 3.2.0

Vantaggi installazione globale:

  • Comando sass disponibile ovunque
  • Ideale per progetti personali e learning
  • Setup veloce

Opzione 2: Installazione Locale (progetto specifico)

# Naviga nella cartella del progetto
cd /path/to/your/project

# Inizializza npm (se non esiste package.json)
npm init -y

# Installa SASS come dev dependency
npm install --save-dev sass

# Verifica installazione
npx sass --version

Vantaggi installazione locale:

  • Versione SASS specifica per progetto
  • Condivisibile con team (via package.json)
  • Migliore per progetti professionali

5. Primo File SCSS

5.1 Creare File SCSS

Crea una cartella per il progetto e un file SCSS:

# Struttura cartelle
my-sass-project/
├── scss/
│   └── style.scss
└── css/
    └── (qui finirà il CSS compilato)

Contenuto di scss/style.scss:

// File: scss/style.scss
// Il mio primo file SASS!

// Variabile per il colore primario
$primary-color: #3498db;
$secondary-color: #2ecc71;
$font-stack: 'Helvetica Neue', Arial, sans-serif;

// Reset base
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

// Stili body
body {
  font-family: $font-stack;
  background: #f5f5f5;
  color: #333;
}

// Header con nesting
.header {
  background: $primary-color;
  color: white;
  padding: 20px;
  
  h1 {
    font-size: 2rem;
    margin-bottom: 10px;
  }
  
  p {
    font-size: 1rem;
    opacity: 0.9;
  }
}

// Button con function
.button {
  background: $primary-color;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
  
  &:hover {
    // darken() è una function SASS built-in
    background: darken($primary-color, 10%);
  }
}

5.2 Compilare SCSS in CSS

Per convertire il file SCSS in CSS utilizzabile dal browser:

# Sintassi base
sass input.scss output.css

# Esempio concreto
sass scss/style.scss css/style.css

# Output nel terminale:
# Compiled scss/style.scss to css/style.css.

Il file css/style.css generato sarà:

/* File: css/style.css (generato automaticamente) */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

body {
  font-family: "Helvetica Neue", Arial, sans-serif;
  background: #f5f5f5;
  color: #333;
}

.header {
  background: #3498db;
  color: white;
  padding: 20px;
}
.header h1 {
  font-size: 2rem;
  margin-bottom: 10px;
}
.header p {
  font-size: 1rem;
  opacity: 0.9;
}

.button {
  background: #3498db;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}
.button:hover {
  background: #2475a8; /* colore calcolato automaticamente! */
}

6. Watch Mode - Compilazione Automatica

Il watch mode monitora i file SCSS e li ricompila automaticamente ad ogni modifica. Essenziale durante lo sviluppo!

# Attiva watch mode
sass --watch scss/style.scss css/style.css

# Output:
# Sass is watching for changes. Press Ctrl-C to stop.
# Compiled scss/style.scss to css/style.css.

# Ora ogni volta che salvi style.scss, viene ricompilato automaticamente!

Per monitorare un'intera cartella:

# Watch su cartella intera
sass --watch scss:css

# Monitora tutti i file .scss nella cartella scss/
# e li compila nella cartella css/

7. Source Maps

Le source maps permettono al browser di mostrare il file SCSS originale negli strumenti di sviluppo, invece del CSS compilato. Utilissimo per il debugging!

# Compilazione con source map (default)
sass scss/style.scss css/style.css

# Disabilita source map
sass --no-source-map scss/style.scss css/style.css

# Con watch e source map
sass --watch scss:css

Con source maps abilitato, nel browser vedrai:

/* Invece di vedere: */
style.css:25

/* Vedrai: */
style.scss:15  ← file e riga originali!

8. Stili di Output CSS

SASS può generare CSS in diversi formati:

Stile Descrizione Uso
expanded (default) CSS leggibile con indentazione Sviluppo
compressed CSS minificato su una riga Produzione 
# Stile expanded (default) - leggibile
sass scss/style.scss css/style.css

# Stile compressed - minificato per produzione
sass --style=compressed scss/style.scss css/style.min.css

# Risultato compressed:
# *{margin:0;padding:0;box-sizing:border-box}body{font-family:"Helvetica Neue",Arial,sans-serif;background:#f5f5f5}...

10. Risoluzione Problemi Comuni

Problema Soluzione
sass: command not found SASS non installato. Esegui npm install -g sass
Error: Can't find stylesheet to import Percorso import errato. Verifica i path relativi
CSS non si aggiorna nel browser Svuota cache browser (Ctrl+F5) o attiva hard reload
Watch mode non funziona Riavvia il comando watch o verifica i permessi sui file

Punti Chiave

  • SASS è un preprocessore che estende CSS con funzionalità di programmazione
  • SCSS è la sintassi raccomandata (compatibile con CSS)
  • Installazione: npm install -g sass
  • Compilazione: sass input.scss output.css
  • Watch mode: sass --watch scss:css
  • Source maps per debugging nei DevTools
  • Stile compressed per produzione
