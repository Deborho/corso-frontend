Lezione 5.1 - Introduzione e Setup JavaScript

Cos'è JavaScript e come integrarlo nelle pagine web

1. Cos'è JavaScript?

JavaScript è un linguaggio di programmazione interpretato che permette di rendere le pagine web interattive e dinamiche. È l'unico linguaggio di programmazione che può essere eseguito nativamente nei browser web.

A cosa serve JavaScript:

  • Interattività: Rispondere a click, input utente, scroll, etc.
  • Manipolazione DOM: Modificare contenuto e struttura della pagina
  • Validazione Form: Controllare dati prima dell'invio
  • Animazioni: Creare effetti visivi e transizioni
  • Comunicazione Server: Inviare e ricevere dati (AJAX)
  • Single Page Applications: Applicazioni web complesse e dinamiche

JavaScript oggi:

  • Nel Browser: Manipola pagine web, gestisce eventi utente
  • Nel Server: Node.js permette di usare JavaScript lato server
  • Mobile: Applicazioni mobili ibride
  • Desktop: Electron per applicazioni desktop

2. JavaScript nel Browser vs Node.js

JavaScript nel Browser JavaScript con Node.js
  • Eseguito nel browser (Chrome, Firefox, etc.)
  • Accede al DOM (Document Object Model)
  • Gestisce eventi utente (click, input, etc.)
  • Interagisce con API del browser (localStorage, fetch)
  • Limitato per sicurezza (no accesso file system)
  • Eseguito sul server
  • Accede a file system e database
  • Gestisce richieste HTTP
  • Può creare server web
  • Nessuna limitazione di sicurezza browser

In questa lezione ci concentreremo su JavaScript nel browser, che è fondamentale per lo sviluppo frontend.

3. Come Includere JavaScript

Esistono tre modi per includere JavaScript in una pagina HTML:

3.1 Inline JavaScript

<button onclick="alert('Cliccato!')">Clicca qui</button>

Quando usarlo: Mai in produzione. Solo per test rapidi.

Svantaggi: Mescola HTML e JavaScript, difficile da mantenere, non scalabile.

3.2 Internal JavaScript

<!DOCTYPE html>
<html>
<head>
    <title>Pagina con JavaScript</title>
</head>
<body>
    <h1 id="titolo">Titolo Iniziale</h1>
    <button id="btn">Cambia Titolo</button>
    
    <script>
        // JavaScript interno
        const btn = document.getElementById('btn');
        const titolo = document.getElementById('titolo');
        
        btn.addEventListener('click', function() {
            titolo.textContent = 'Titolo Cambiato!';
        });
    </script>
</body>
</html>

Quando usarlo: Per piccole pagine o demo. JavaScript specifico per quella pagina.

Vantaggi: Tutto in un file, facile da visualizzare.

Svantaggi: Non riutilizzabile, file HTML diventa grande.

3.3 External JavaScript (Consigliato)

<!-- File: index.html -->
<!DOCTYPE html>
<html>
<head>
    <title>Pagina con JavaScript</title>
</head>
<body>
    <h1 id="titolo">Titolo Iniziale</h1>
    <button id="btn">Cambia Titolo</button>
    
    <!-- Script esterno caricato alla fine del body -->
    <script src="script.js"></script>
</body>
</html>
// File: script.js
const btn = document.getElementById('btn');
const titolo = document.getElementById('titolo');

btn.addEventListener('click', function() {
    titolo.textContent = 'Titolo Cambiato!';
});

Quando usarlo: Sempre in progetti reali.

Vantaggi:

  • Separa HTML e JavaScript (separation of concerns)
  • Codice riutilizzabile su più pagine
  • Migliore organizzazione del progetto
  • File JavaScript può essere cachato dal browser

Dove posizionare il tag script:

  • Alla fine del body (consigliato): Il DOM è già caricato quando JavaScript viene eseguito
  • Nell'head con defer: <script src="script.js" defer></script> - Scarica in parallelo ma esegue dopo il DOM
  • Nell'head con async: <script src="script.js" async></script> - Per script che non dipendono dal DOM

4. Console del Browser

La console è lo strumento principale per sviluppare e debuggare JavaScript. Permette di eseguire codice JavaScript e visualizzare output, errori e messaggi di log.

Come aprire la console:

  • Chrome/Edge: F12 o Ctrl+Shift+J (Windows/Linux), Cmd+Option+J (Mac)
  • Firefox: F12 o Ctrl+Shift+K (Windows/Linux), Cmd+Option+K (Mac)
  • Safari: Cmd+Option+C (Mac)

4.1 console.log()

// Stampare messaggi nella console
console.log('Ciao JavaScript!');

// Stampare variabili
const nome = 'Mario';
console.log(nome);  // Output: Mario

// Stampare più valori
const età = 25;
console.log('Nome:', nome, 'Età:', età);  // Output: Nome: Mario Età: 25

// Stampare oggetti
const persona = { nome: 'Mario', età: 25 };
console.log(persona);  // Output: {nome: "Mario", età: 25}

4.2 Altri Metodi Console Utili

// console.error() - Mostra errori in rosso
console.error('Questo è un errore!');

// console.warn() - Mostra avvisi in giallo
console.warn('Questo è un avviso!');

// console.table() - Mostra array/oggetti come tabella
const utenti = [
    { nome: 'Mario', età: 25 },
    { nome: 'Luigi', età: 30 }
];
console.table(utenti);

// console.clear() - Pulisce la console
console.clear();

5. Commenti in JavaScript

// Questo è un commento su singola riga
// I commenti non vengono eseguiti

/* 
   Questo è un commento
   su più righe
   utile per spiegazioni lunghe
*/

const nome = 'Mario';  // Commento alla fine della riga

/*
 * Commento di documentazione
 * @param {string} nome - Il nome dell'utente
 * @returns {string} - Il messaggio di saluto
 */
function saluta(nome) {
    return `Ciao ${nome}!`;
}

6. Statement e Punto e Virgola

In JavaScript, ogni istruzione (statement) può terminare con un punto e virgola ;

Il punto e virgola è opzionale grazie all'Automatic Semicolon Insertion (ASI), ma è considerata best practice usarlo sempre per evitare bug.

// Con punto e virgola (consigliato)
const nome = 'Mario';
const età = 25;
console.log(nome);

// Senza punto e virgola (funziona ma sconsigliato)
const nome = 'Mario'
const età = 25
console.log(nome)

// Casi dove il punto e virgola è necessario
const a = 1; const b = 2;  // Due statement sulla stessa riga

// Caso problematico senza punto e virgola
const saluto = 'Ciao'
[1, 2, 3].forEach(n => console.log(n))
// JavaScript interpreta come: const saluto = 'Ciao'[1, 2, 3].forEach(...)

7. Case Sensitivity

JavaScript è case-sensitive: distingue tra maiuscole e minuscole.

// Queste sono variabili diverse
const nome = 'Mario';
const Nome = 'Luigi';
const NOME = 'Giovanna';

console.log(nome);  // Mario
console.log(Nome);  // Luigi
console.log(NOME);  // Giovanna

// I nomi delle funzioni sono case-sensitive
console.log('test');  // Funziona
Console.log('test');  // ERRORE: Console is not defined

8. Convenzioni di Naming

camelCase è la convenzione standard per JavaScript:

// camelCase per variabili e funzioni (consigliato)
const nomeUtente = 'Mario';
const numeroTelefono = '1234567890';
const isAttivo = true;

function calcolaSomma(a, b) {
    return a + b;

// PascalCase per classi e costruttori

    constructor(nome, cognome) {
        this.nome = nome;
        this.cognome = cognome;
    }

// SCREAMING_SNAKE_CASE per costanti globali

// Nomi descrittivi

9. Esempio Pratico Completo

Esempio di una semplice pagina con JavaScript:

Titolo Originale

Questo è un paragrafo.

<h3 id="titolo">Titolo Originale</h3>
<p id="paragrafo">Questo è un paragrafo.</p>
<button id="btn-cambia">Cambia Contenuto</button>
<button id="btn-reset">Reset</button>

<script>
    // Seleziona elementi dal DOM
    const titolo = document.getElementById('titolo');
    const paragrafo = document.getElementById('paragrafo');
    const btnCambia = document.getElementById('btn-cambia');
    const btnReset = document.getElementById('btn-reset');
    
    // Memorizza contenuto originale
    const titoloOriginale = titolo.textContent;
    const paragrafoOriginale = paragrafo.textContent;
    
    // Aggiungi event listener al bottone
    btnCambia.addEventListener('click', function() {
        console.log('Bottone cliccato!');
        titolo.textContent = 'Nuovo Titolo!';
        paragrafo.textContent = 'Il contenuto è stato modificato con JavaScript.';
    });
    
    // Reset contenuto
    btnReset.addEventListener('click', function() {
        console.log('Reset cliccato!');
        titolo.textContent = titoloOriginale;
        paragrafo.textContent = paragrafoOriginale;
    });
</script>

10. Note Importanti

  • JavaScript è interpretato: Non serve compilazione, il browser esegue il codice direttamente
  • Usa sempre const/let: Non usare var nelle versioni moderne di JavaScript
  • Semicolons: Usa sempre il punto e virgola per evitare bug
  • Console: Usa console.log() costantemente per debuggare
  • DevTools: Impara a usare gli strumenti di sviluppo del browser
  • Errori: Leggi sempre gli errori nella console, indicano il problema e la riga

11. Preparazione per TypeScript e React

Perché iniziare con setup corretto è importante:

  • React usa JSX: Sintassi simile a HTML dentro JavaScript
  • TypeScript richiede setup: Compila TypeScript in JavaScript
  • Moduli ES6: React e TypeScript si basano su import/export
  • DevTools: React ha estensioni browser dedicate
  • Console: Fondamentale per debuggare componenti React

Padroneggiare questi concetti base rende molto più semplice l'apprendimento di strumenti avanzati.
