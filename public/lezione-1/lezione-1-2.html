Lezione 1.2 - Form HTML

Elementi e componenti dei moduli web

Nota per la versione stampata: Questa lezione contiene esempi interattivi visualizzabili tramite iframe. Gli iframe non possono essere stampati per limitazioni tecniche dei browser. Per visualizzare gli esempi completi, consultare la versione online di questo documento. I percorsi degli esempi sono indicati nelle sezioni corrispondenti.

1. Cosa sono i Form HTML

I form HTML (moduli o moduli di input) sono sezioni di una pagina web che permettono agli utenti di inserire e inviare dati al server. Sono lo strumento principale per l'interazione tra l'utente e il sito web.

I form vengono utilizzati per molteplici scopi:

  • Registrazione e login degli utenti
  • Ricerca di contenuti
  • Invio di messaggi e contatti
  • Compilazione di questionari e sondaggi
  • Acquisti online (e-commerce)
  • Caricamento di file
  • Inserimento di commenti

Un form è delimitato dal tag <form> e contiene al suo interno vari elementi di input che raccolgono i dati dall'utente. Quando l'utente invia il form (solitamente tramite un pulsante), i dati vengono trasmessi al server per essere elaborati.

2. Il Tag <form>

Il tag <form> è il contenitore principale che raggruppa tutti gli elementi di input. Ha diversi attributi importanti:

  • action: specifica l'URL del server dove inviare i dati del form
  • method: definisce il metodo HTTP da utilizzare (GET o POST)
  • enctype: specifica come codificare i dati del form (necessario per caricare file)
  • autocomplete: abilita o disabilita l'autocompletamento dei campi
  • novalidate: disabilita la validazione automatica HTML5
  • target: specifica dove visualizzare la risposta (_self, _blank, ecc.)
<form action="/submit" method="POST">
    <!-- Elementi del form -->
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome" required>
    
    <button type="submit">Invia</button>
</form>

3. Elementi dei Form

I form HTML sono composti da vari elementi che permettono l'inserimento e la selezione di dati. Ecco l'elenco completo degli elementi disponibili:

3.1 <input>

L'elemento più versatile e utilizzato. Il comportamento cambia in base all'attributo type:

  • type="text" - campo di testo semplice
  • type="password" - campo per password (caratteri nascosti)
  • type="email" - campo per indirizzi email (con validazione)
  • type="tel" - campo per numeri di telefono
  • type="url" - campo per indirizzi web
  • type="number" - campo per valori numerici
  • type="range" - selettore a cursore per valori numerici
  • type="date" - selettore di data
  • type="time" - selettore di orario
  • type="datetime-local" - selettore di data e ora
  • type="month" - selettore di mese
  • type="week" - selettore di settimana
  • type="color" - selettore di colore
  • type="search" - campo di ricerca
  • type="checkbox" - casella di spunta (selezione multipla)
  • type="radio" - pulsante radio (selezione singola)
  • type="file" - selettore di file
  • type="submit" - pulsante di invio
  • type="reset" - pulsante di reset
  • type="button" - pulsante generico
  • type="hidden" - campo nascosto
<!-- Testo -->
<input type="text" id="username" name="username" 
       placeholder="Inserisci username" 
       maxlength="20" required>

<!-- Password -->
<label for="password">Password:</label>
<input type="password" id="password" name="password" 
       minlength="8" required>

<!-- Email -->
<label for="email">Email:</label>
<input type="email" id="email" name="email" 
       placeholder="nome@esempio.com" required>

<!-- Telefono -->
<label for="tel">Telefono:</label>
<input type="tel" id="tel" name="tel" 
       pattern="[0-9]{10}" placeholder="1234567890">

<!-- URL -->
<label for="website">Sito Web:</label>
<input type="url" id="website" name="website" 
       placeholder="https://esempio.com">

3.2 <textarea>

Area di testo multiriga per l'inserimento di contenuti più lunghi come commenti, descrizioni o messaggi.

A differenza di <input type="text">, permette di inserire testo su più righe e ha dimensioni ridimensionabili.

<label for="messaggio">Messaggio:</label>
<textarea id="messaggio" name="messaggio" 
          rows="5" cols="50" 
          placeholder="Scrivi qui il tuo messaggio..."
          maxlength="500"></textarea>

3.3 <select>

Crea un menu a tendina (dropdown) per selezionare una o più opzioni da un elenco predefinito.

Gli elementi interni <option> rappresentano le voci selezionabili. È possibile raggruppare le opzioni usando <optgroup>.

<label for="citta">Città:</label>
<select id="citta" name="citta" required>
    <option value="">Seleziona una città</option>
    <option value="roma">Roma</option>
    <option value="milano">Milano</option>
    <option value="napoli">Napoli</option>
</select>

<!-- Select con gruppi -->
<select id="paese" name="paese">
    <optgroup label="Europa">
        <option value="it">Italia</option>
        <option value="fr">Francia</option>
        <option value="de">Germania</option>
    </optgroup>
    <optgroup label="America">
        <option value="us">Stati Uniti</option>
        <option value="ca">Canada</option>
    </optgroup>
</select>

<!-- Select multipla -->
<select id="hobby" name="hobby" multiple size="4">
    <option value="sport">Sport</option>
    <option value="musica">Musica</option>
    <option value="lettura">Lettura</option>
    <option value="cinema">Cinema</option>
</select>

3.4 Checkbox

Le checkbox permettono la selezione multipla di opzioni. Ogni checkbox è indipendente dalle altre e può essere selezionata o deselezionata liberamente.

<fieldset>
    <legend>Seleziona i tuoi interessi:</legend>
    
    <input type="checkbox" id="sport" name="interessi" value="sport">
    <label for="sport">Sport</label>
    
    <input type="checkbox" id="musica" name="interessi" value="musica">
    <label for="musica">Musica</label>
    
    <input type="checkbox" id="viaggi" name="interessi" value="viaggi" checked>
    <label for="viaggi">Viaggi</label>
</fieldset>

3.5 Radio Button

I radio button permettono la selezione di una sola opzione tra diverse disponibili. Tutti i radio button dello stesso gruppo devono avere lo stesso valore per l'attributo name.

<fieldset>
    <legend>Genere:</legend>
    
    <input type="radio" id="maschio" name="genere" value="M" required>
    <label for="maschio">Maschio</label>
    
    <input type="radio" id="femmina" name="genere" value="F">
    <label for="femmina">Femmina</label>
    
    <input type="radio" id="altro" name="genere" value="altro">
    <label for="altro">Altro</label>
</fieldset>

3.6 <button>

Elemento per creare pulsanti interattivi. Il tipo di pulsante è definito dall'attributo type:

  • type="submit" - invia il form
  • type="reset" - resetta tutti i campi del form
  • type="button" - pulsante generico (richiede JavaScript)

A differenza di <input type="submit">, <button> può contenere HTML al suo interno (testo, immagini, icone).

<button type="submit">Invia</button>
<button type="reset">Cancella</button>
<button type="button" onclick="alert('Ciao!')">Cliccami</button>

<!-- Button con contenuto HTML -->
<button type="submit">
    <img src="send-icon.png" alt="">
    Invia Messaggio
</button>

3.7 <label>

Etichetta descrittiva associata a un elemento di input. Migliora l'usabilità e l'accessibilità del form. Quando si clicca sulla label, il focus viene spostato automaticamente sul campo associato.

L'associazione avviene tramite l'attributo for della label e l'attributo id dell'input.

<!-- Metodo 1: associazione tramite for e id -->
<label for="username">Nome Utente:</label>
<input type="text" id="username" name="username">

<!-- Metodo 2: input all'interno della label -->
<label>
    Email:
    <input type="email" name="email">
</label>

3.8 <fieldset>

Raggruppa elementi correlati del form creando un riquadro visivo. Utile per organizzare form complessi in sezioni logiche.

<fieldset>
    <legend>Informazioni Personali</legend>
    
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome">
    
    <label for="cognome">Cognome:</label>
    <input type="text" id="cognome" name="cognome">
</fieldset>

3.9 <legend>

Fornisce un titolo/descrizione per un <fieldset>. Viene visualizzato come intestazione del gruppo.

3.10 <datalist>

Fornisce un elenco di suggerimenti predefiniti per un campo <input>. Combina la flessibilità di un campo di testo con la comodità di un menu a tendina.

L'utente può scegliere un'opzione dall'elenco o digitare un valore personalizzato.

<label for="browser">Scegli un browser:</label>
<input list="browsers" id="browser" name="browser">

<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
    <option value="Opera">
</datalist>

3.11 <output>

Rappresenta il risultato di un calcolo o di un'operazione dell'utente. Viene utilizzato insieme a JavaScript per mostrare dinamicamente dei valori.

<form oninput="risultato.value=parseInt(a.value)+parseInt(b.value)">
    <input type="number" id="a" value="0"> +
    <input type="number" id="b" value="0"> =
    <output name="risultato" for="a b">0</output>
</form>

4. Attributi Comuni degli Elementi di Input

Gli elementi di input condividono molti attributi comuni che ne definiscono il comportamento:

  • name - nome del campo (necessario per l'invio dei dati)
  • id - identificatore univoco (usato per associare label e CSS/JavaScript)
  • value - valore predefinito o corrente del campo
  • placeholder - testo di suggerimento mostrato quando il campo è vuoto
  • required - rende il campo obbligatorio
  • disabled - disabilita il campo (non modificabile e non inviato)
  • readonly - campo di sola lettura (non modificabile ma inviato)
  • autofocus - il campo riceve automaticamente il focus al caricamento della pagina
  • autocomplete - abilita/disabilita l'autocompletamento
  • maxlength - lunghezza massima del testo
  • minlength - lunghezza minima del testo
  • pattern - espressione regolare per validare il formato
  • min / max - valori minimo e massimo per input numerici
  • step - incremento per input numerici
  • multiple - permette selezione multipla (per file e select)
<!-- Campo obbligatorio con lunghezza limitata -->
<input type="text" name="username" 
       required minlength="3" maxlength="20"
       placeholder="Min 3, max 20 caratteri">

<!-- Campo numerico con range -->
<input type="number" name="eta" 
       min="18" max="99" step="1" required>

<!-- Campo con pattern di validazione -->
<input type="text" name="codice" 
       pattern="[A-Z]{3}[0-9]{3}"
       title="Formato: 3 lettere maiuscole + 3 numeri">

<!-- Campo disabilitato -->
<input type="text" value="Non modificabile" disabled>

<!-- Campo di sola lettura -->
<input type="text" value="Sola lettura" readonly>

5. Validazione dei Form

HTML5 offre meccanismi di validazione nativi che permettono di verificare i dati inseriti prima dell'invio al server, migliorando l'esperienza utente e riducendo il carico sul server.

5.1 Validazione Automatica HTML5

Alcuni attributi HTML5 attivano automaticamente la validazione:

  • required - il campo deve essere compilato
  • type="email" - verifica formato email
  • type="url" - verifica formato URL
  • pattern - verifica con espressione regolare personalizzata
  • min, max, minlength, maxlength - vincoli di range

5.2 Attributo novalidate

L'attributo novalidate sul tag <form> disabilita la validazione automatica HTML5, utile quando si vuole implementare una validazione personalizzata con JavaScript.

<form action="/submit" method="POST" novalidate>
    <!-- La validazione HTML5 è disabilitata -->
    <input type="email" name="email" required>
</form>

Nota: Per approfondire come stilizzare i form con le pseudo-classi CSS (:valid, :invalid, ecc.), consulta la Lezione 2.1 - CSS Base.

6. Metodi HTTP: GET vs POST

L'attributo method del form specifica come inviare i dati al server. I due metodi principali sono:

6.1 Metodo GET

  • I dati vengono aggiunti all'URL come parametri query string
  • Visibile nell'URL del browser
  • Limitazione sulla quantità di dati (max ~2048 caratteri)
  • I dati possono essere salvati nei bookmark
  • Usato per ricerche e operazioni che non modificano dati
  • Non sicuro per dati sensibili (password, dati personali)
<form action="/search" method="GET">
    <input type="search" name="q" placeholder="Cerca...">
    <button type="submit">Cerca</button>
</form>

<!-- URL risultante: /search?q=termine+di+ricerca -->

6.2 Metodo POST

  • I dati vengono inviati nel corpo della richiesta HTTP
  • Non visibile nell'URL
  • Nessun limite pratico sulla quantità di dati
  • I dati non vengono salvati nella cronologia del browser
  • Usato per inviare dati sensibili o modificare dati sul server
  • Necessario per upload di file
<form action="/register" method="POST">
    <input type="email" name="email" required>
    <input type="password" name="password" required>
    <button type="submit">Registrati</button>
</form>

7. Upload di File

Per permettere agli utenti di caricare file, è necessario:

  • Usare <input type="file">
  • Impostare method="POST" nel form
  • Impostare enctype="multipart/form-data" nel form
<form action="/upload" method="POST" enctype="multipart/form-data">
    <label for="file">Seleziona un file:</label>
    <input type="file" id="file" name="file" required>
    
    <!-- Upload multiplo -->
    <label for="files">Seleziona più file:</label>
    <input type="file" id="files" name="files" multiple>
    
    <!-- Limitare i tipi di file -->
    <label for="image">Seleziona un'immagine:</label>
    <input type="file" id="image" name="image" 
           accept="image/png, image/jpeg">
    
    <button type="submit">Carica</button>
</form>

8. Best Practices per i Form

  • Usa sempre label per gli input - migliora accessibilità e usabilità
  • Associa correttamente label e input - usa attributi for e id
  • Fornisci feedback visivo - usa :valid e :invalid per mostrare lo stato
  • Usa placeholder in modo appropriato - non sostituisce le label
  • Raggruppa campi correlati - usa <fieldset> e <legend>
  • Indica i campi obbligatori - visivamente e con l'attributo required
  • Valida sia lato client che server - la validazione client è solo un aiuto all'utente
  • Usa i type corretti - migliora UX su mobile (tastiere contestuali)
  • Fornisci messaggi di errore chiari - usa l'attributo title o JavaScript
  • Testa l'accessibilità - assicurati che il form sia navigabile da tastiera
  • Previeni invii multipli - disabilita il pulsante dopo il primo invio
  • Proteggi i dati sensibili - usa sempre HTTPS per form con dati sensibili

10. Attributo autocomplete

L'attributo autocomplete aiuta il browser a compilare automaticamente i campi con valori salvati in precedenza. HTML5 definisce valori standardizzati per questo attributo:

  • name - nome completo
  • given-name - nome
  • family-name - cognome
  • email - indirizzo email
  • username - username
  • new-password - nuova password (registrazione)
  • current-password - password corrente (login)
  • tel - telefono
  • street-address - indirizzo
  • country - paese
  • postal-code - codice postale
  • cc-number - numero carta di credito
  • bday - data di nascita

Riepilogo

I form HTML sono essenziali per creare siti web interattivi. I punti chiave da ricordare:

  • Il tag <form> contiene tutti gli elementi di input
  • Esistono molti tipi di input per diversi scopi
  • Usa sempre <label> per migliorare accessibilità e UX
  • HTML5 offre validazione nativa tramite attributi specifici
  • POST è preferibile a GET per dati sensibili
  • Raggruppa campi correlati con <fieldset>
  • Testa sempre l'accessibilità e la navigazione da tastiera
