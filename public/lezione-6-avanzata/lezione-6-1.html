3. Installare TypeScript

Requisiti: Node.js e npm installati sul sistema.

// Installazione globale
npm install -g typescript

// Verifica installazione
tsc --version

// Output: Version 5.3.3 (o superiore)
// Installazione locale in un progetto
npm init -y
npm install --save-dev typescript

// Verifica
npx tsc --version

4. Primo File TypeScript

I file TypeScript hanno estensione .ts. Vengono compilati in .js con il comando tsc.

Esempio: hello.ts

// hello.ts
function saluta(nome: string): string {
    return `Ciao, ${nome}!`;
}

const messaggio: string = saluta("TypeScript");
console.log(messaggio);
// Compilazione
tsc hello.ts

// Genera hello.js:
function saluta(nome) {
    return "Ciao, " + nome + "!";
}
var messaggio = saluta("TypeScript");
console.log(messaggio);

// Esecuzione
node hello.js
// Output: Ciao, TypeScript!

5. File di Configurazione: tsconfig.json

tsconfig.json definisce come TypeScript compila il progetto.

// Genera tsconfig.json
tsc --init

tsconfig.json Base

{
  "compilerOptions": {
    /* Language and Environment */
    "target": "ES2020",                  // Versione JavaScript output
    
    /* Modules */
    "module": "ESNext",                  // Sistema moduli (ESNext, CommonJS)
    "moduleResolution": "node",          // Risoluzione moduli come Node.js
    
    /* Emit */
    "outDir": "./dist",                  // Cartella output file compilati
    "rootDir": "./src",                  // Cartella sorgenti TypeScript
    "removeComments": true,              // Rimuovi commenti dal .js
    
    /* Type Checking */
    "strict": true,                      // Abilita tutte le opzioni strict
    "noImplicitAny": true,               // Errore se TypeScript inferisce 'any'
    "strictNullChecks": true,            // null/undefined devono essere gestiti
    "strictFunctionTypes": true,         // Controlli pi√π rigidi sulle funzioni
    
    /* Interop Constraints */
    "esModuleInterop": true,             // Compatibilit√† import CommonJS
    "forceConsistentCasingInFileNames": true, // Case-sensitive nei nomi file
    
    /* Skip Checking */
    "skipLibCheck": true                 // Salta controllo file .d.ts di librerie
  },
  "include": ["src/**/*"],               // File da compilare
  "exclude": ["node_modules", "dist"]    // File da escludere
}

6. Opzioni Compiler Importanti

  • target: Versione JavaScript output (ES5, ES2015, ES2020, ESNext)
  • module: Sistema moduli (CommonJS per Node, ESNext per browser moderni)
  • strict: Abilita tutti i controlli strict (raccomandato)
  • outDir: Dove salvare i file .js compilati
  • sourceMap: Genera .map files per debugging
  • lib: Librerie TypeScript da includere (DOM, ES2020, ecc.)
  • noEmitOnError: Non genera .js se ci sono errori TypeScript

8. Watch Mode

Watch mode ricompila automaticamente quando modifichi i file:

// Watch mode per singolo file
tsc hello.ts --watch

// Watch mode per intero progetto
tsc --watch

// Output:
// [12:34:56 PM] Starting compilation in watch mode...
// [12:34:57 PM] Found 0 errors. Watching for file changes.

9. TypeScript nel Browser

Per usare TypeScript nel browser:

  1. Compila TypeScript in JavaScript
  2. Includi il file .js nell'HTML
  3. Oppure usa bundler come Vite o Webpack

Workflow Base

// 1. Scrivi TypeScript (src/app.ts)
const messaggio: string = "Hello TypeScript!";
console.log(messaggio);

// 2. Compila
tsc

// 3. Include nell'HTML
<script src="dist/app.js"></script>

11. Best Practices per Iniziare

  • Evita any: Specifica sempre tipi espliciti, evita any
  • Type Inference: Fidati dell'inferenza quando ovvia, esplicita quando serve chiarezza
  • Incrementale: Puoi migrare JavaScript ‚Üí TypeScript gradualmente (.js e .ts convivono)
  • Lint: Usa ESLint con plugin TypeScript per mantenere codice pulito
  • Documentazione: Consulta la TypeScript Handbook
