Lezione 4.1 - INSERT INTO

Inserimento di dati nelle tabelle

1. INSERT INTO

Il comando INSERT INTO viene utilizzato per inserire nuove righe (record) in una tabella. È possibile inserire una singola riga o multiple righe contemporaneamente.

1.1 Sintassi con tutte le colonne

INSERT INTO nome_tabella
VALUES (valore1, valore2, valore3, ...);

1.2 Sintassi con colonne specifiche

INSERT INTO nome_tabella (colonna1, colonna2, colonna3)
VALUES (valore1, valore2, valore3);

2. Esempi di INSERT

2.1 Inserimento Singolo

-- Inserisci un nuovo cliente
INSERT INTO clienti (nome, cognome, email)
VALUES ('Mario', 'Rossi', 'mario.rossi@email.com');

Risultato:

Query OK, 1 row affected (0.01 sec)

2.2 Inserimento con AUTO_INCREMENT

Se la colonna ID ha AUTO_INCREMENT, non è necessario specificarla:

-- L'ID viene generato automaticamente
INSERT INTO clienti (nome, cognome, email, telefono)
VALUES ('Giulia', 'Bianchi', 'giulia.bianchi@email.com', '333-1234567');

2.3 Inserimento Multiple Righe

-- Inserisci più clienti in una sola query
INSERT INTO clienti (nome, cognome, email) VALUES
    ('Luca', 'Verdi', 'luca.verdi@email.com'),
    ('Anna', 'Neri', 'anna.neri@email.com'),
    ('Paolo', 'Gialli', 'paolo.gialli@email.com');

Risultato:

Query OK, 3 rows affected (0.02 sec)
Records: 3  Duplicates: 0  Warnings: 0

3. Inserimento con Valori Default

Se una colonna ha un valore DEFAULT, puoi ometterla nell'INSERT oppure usare la keyword DEFAULT.

-- La colonna "stato" userà il valore default
INSERT INTO clienti (nome, cognome, email)
VALUES ('Sara', 'Blu', 'sara.blu@email.com');

-- Oppure esplicitamente
INSERT INTO clienti (nome, cognome, email, stato)
VALUES ('Marco', 'Rosa', 'marco.rosa@email.com', DEFAULT);

5. Esempio Finale

Esempio: Popolare il database e-commerce

-- Assicurati di essere nel database corretto
USE ecommerce;

-- Inserisci i clienti
INSERT INTO clienti (nome, cognome, email, telefono) VALUES
    ('Mario', 'Rossi', 'mario.rossi@email.com', '333-1111111'),
    ('Giulia', 'Bianchi', 'giulia.bianchi@email.com', '333-2222222'),
    ('Luca', 'Verdi', 'luca.verdi@email.com', '333-3333333'),
    ('Anna', 'Neri', 'anna.neri@email.com', '333-4444444'),
    ('Paolo', 'Gialli', 'paolo.gialli@email.com', '333-5555555');

-- Inserisci i prodotti
INSERT INTO prodotti (nome, descrizione, prezzo, quantita, categoria) VALUES
    ('iPhone 15', 'Smartphone Apple ultima generazione', 1199.00, 25, 'Smartphone'),
    ('Samsung Galaxy S24', 'Smartphone Samsung top di gamma', 999.00, 40, 'Smartphone'),
    ('MacBook Air', 'Laptop Apple M2', 1299.00, 15, 'Computer'),
    ('iPad Pro', 'Tablet Apple 12.9 pollici', 1099.00, 20, 'Tablet'),
    ('AirPods Pro', 'Auricolari wireless Apple', 279.00, 100, 'Accessori');

-- Inserisci alcuni ordini
INSERT INTO ordini (cliente_id, totale, stato) VALUES
    (1, 1199.00, 'completato'),
    (2, 1578.00, 'in spedizione'),
    (3, 279.00, 'in elaborazione'),
    (1, 999.00, 'completato'),
    (4, 2398.00, 'in spedizione');

-- Verifica gli inserimenti
SELECT * FROM clienti;
SELECT * FROM prodotti;
SELECT * FROM ordini;

Note importanti:

  • I valori stringa devono essere racchiusi tra apici singoli ' '
  • I numeri non richiedono apici
  • NULL può essere inserito per valori mancanti (se la colonna lo permette)
  • Le date devono essere nel formato 'YYYY-MM-DD'
