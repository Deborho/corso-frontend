8.9 - .gitignore e File Management

Cos'√® .gitignore?

Il file .gitignore specifica quali file e directory Git deve ignorare (non tracciare).

Perch√© usare .gitignore?

‚úÖ Evita di committare file generati (build/, dist/)
‚úÖ Non traccia dipendenze (node_modules/, vendor/)
‚úÖ Protegge file sensibili (.env, secrets.json)
‚úÖ Ignora file di sistema (.DS_Store, Thumbs.db)
‚úÖ Esclude file IDE (.vscode/, .idea/)
‚úÖ Mantiene repository pulito e leggero

Dove va .gitignore?
- Root del repository
- Nome esatto: .gitignore (con punto)
- Committato nel repository
- Condiviso con team

Sintassi .gitignore

Pattern Base:

# Directory (con trailing slash):

Pattern Matching Avanzati

Wildcard:

*        Match qualsiasi carattere (eccetto /)
?        Match singolo carattere
[abc]    Match uno dei caratteri tra parentesi
[0-9]    Match range di caratteri
**       Match zero o pi√π directory

Esempi:

# Directory dist ovunque:

# Ignora src/temp ma non other/temp:

Global .gitignore

File ignorati per TUTTI i tuoi repository locali:

# 1. Crea file gitignore globale:
touch ~/.gitignore_global

# 2. Configura Git per usarlo:
git config --global core.excludesfile ~/.gitignore_global

# 3. Aggiungi patterns comuni:
cat <<EOF >> ~/.gitignore_global
# OS
.DS_Store
Thumbs.db
desktop.ini

# Editors
.vscode/
.idea/
*.swp
*.swo
*~

# Misc
.temp/
*.bak
EOF

# Ora questi file sono ignorati in TUTTI i repository!

Gestire File Gi√† Tracciati

# Problema: file gi√† committato, ora vuoi ignorarlo

# ‚ö†Ô∏è Non basta! File gi√† tracciato continua ad esserlo

# Soluzione: Rimuovi da tracking (mantieni locale):

# Ora node_modules/ √® ignorato

# Per singolo file:

# ‚ö†Ô∏è --cached = rimuovi solo da Git, non da filesystem
# ‚ö†Ô∏è senza --cached = rimuove anche da filesystem

# Verifica file ignorati:

# Forza aggiungere file ignorato (raramente necessario):

‚úÖ Best Practices - .gitignore

‚úÖ DO - Fare:

  • Crea .gitignore PRIMA del primo commit
  • Usa template per la tua tecnologia (gitignore.io)
  • Ignora sempre node_modules/, build/, dist/
  • Ignora sempre .env e file con segreti
  • Ignora file OS (.DS_Store, Thumbs.db)
  • Commita .gitignore nel repository
  • Usa global .gitignore per preferenze personali
  • Documenta perch√© ignori file specifici (commenti)

‚ùå DON'T - Evitare:

  • Non committare .env, secrets, passwords
  • Non committare node_modules/ o vendor/
  • Non committare file build/compilati
  • Non ignorare .gitignore stesso
  • Non usare .gitignore per file-specific (usa .git/info/exclude)
  • Non committare file IDE di team members

File Gi√† Committati con Secrets

‚ö†Ô∏è Se hai committato file con password/keys per errore:

Problema GRAVE:

git commit -m "Add config"

# ‚ùå CHIAVE COMPROMESSA!
# Anche se rimuovi file, rimane nella storia Git

Soluzione Immediata:
1. REVOCA la chiave/password (cambiale!)
2. Rimuovi file dalla storia Git (BFG Repo Cleaner o git filter-branch)
3. Force push (‚ö†Ô∏è impatta collaboratori)

Prevenzione:
# 1. Usa .env.example con valori finti:

DB_PASSWORD=your_password_here

# 2. Ignora .env reale:
echo ".env" >> .gitignore

# 3. Documenta in README:
"Copy .env.example to .env and fill with real values"

# 4. Usa tools come git-secrets per prevenire:
git secrets --install
git secrets --register-aws
