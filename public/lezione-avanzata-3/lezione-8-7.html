Clonare un Repository

# Clone con nome directory custom:

Gestire Remote

# Lista remotes:
git remote
# Output: origin  (nome default dopo clone)

# Lista remotes con URL:
git remote -v
# Output:
# origin  https://github.com/user/repo.git (fetch)
# origin  https://github.com/user/repo.git (push)

# Aggiungere remote:
git remote add nome-remote https://github.com/user/repo.git
git remote add upstream https://github.com/original/repo.git

# Rinominare remote:
git remote rename origin old-origin

# Rimuovere remote:
git remote remove nome-remote
git remote rm nome-remote

# Vedere info dettagliate remote:
git remote show origin
# Mostra: URL, branch tracked, configurazione push/pull

# Cambiare URL remote:
git remote set-url origin https://github.com/newuser/repo.git

Push - Inviare Commit al Remote

# Push branch corrente al remote:
git push origin main

# Push e imposta upstream (tracking):
git push -u origin main
git push --set-upstream origin main
# Dopo questo, basta fare: git push

# Push tutti i branch:
git push --all origin

# Push con tags:
git push --tags

# Push branch specifico:
git push origin feature-auth

# Cancellare branch remoto:
git push origin --delete feature-auth
git push origin :feature-auth  # Sintassi vecchia

# Force push (‚ö†Ô∏è PERICOLOSO):
git push --force origin main
git push -f origin main
# Riscrive storia remota! Usa solo se necessario!

# Force push sicuro (fallisce se altri hanno pushato):
git push --force-with-lease origin main

Pull - Scaricare Aggiornamenti

# Pull (fetch + merge):
git pull origin main

# Pull con rebase invece di merge:
git pull --rebase origin main

# Pull da upstream tracking branch:
git pull
# Funziona se hai configurato upstream con -u

git pull = git fetch + git merge

# Equivalente manuale:
git fetch origin
git merge origin/main

# Pull all remotes:
git pull --all

# ‚ö†Ô∏è Pull pu√≤ causare merge conflicts
# Risolvi come normale merge conflict

Fetch - Scaricare Senza Merge

# Fetch da remote (scarica ma non merge):
git fetch origin

# Fetch branch specifico:
git fetch origin main

# Fetch tutti i remotes:
git fetch --all

# Fetch e cancella branch remoti eliminati:
git fetch --prune
git fetch -p

# Dopo fetch, vedi differenze:
git log origin/main..main      # Commit locali non pushati
git log main..origin/main      # Commit remoti non ancora merged

# Merge manuale dopo fetch:
git merge origin/main

Differenza Fetch vs Pull:
git fetch    # Scarica, NON modifica working directory (sicuro)
git pull     # Scarica E merge (pu√≤ creare conflitti)

# Best practice: fetch prima, poi merge se ok

Tracking Branches

# Vedere tracking relationships:
git branch -vv
# Output:
# * main    7a3f2d1 [origin/main] feat: dark mode
#   feature 2b5e9f3 feat: login

# Impostare upstream per branch:
git branch --set-upstream-to=origin/main main
git branch -u origin/main main

# Creare branch locale da remote:
git checkout -b feature origin/feature
# Oppure (Git 2.23+):
git switch -c feature origin/feature

# Automatico con stesso nome:
git checkout feature
# Git crea automaticamente local branch tracking origin/feature

# Vedere branch remoti:
git branch -r
# Output:
#   origin/main
#   origin/feature-auth
#   origin/develop

‚úÖ Best Practices - Remote

‚úÖ DO - Fare:

  • Usa SSH per repository che usi frequentemente
  • Pull prima di iniziare a lavorare (evita conflitti)
  • Push spesso (backup e collaborazione)
  • Usa branch per features, non committare su main
  • Usa fetch prima di pull per vedere cosa cambier√†
  • Configura upstream con -u al primo push
  • Usa --force-with-lease invece di --force

‚ùå DON'T - Evitare:

  • Non fare force push su branch condivisi (main, develop)
  • Non committare credenziali, password, API keys
  • Non ignorare conflitti di merge
  • Non pushare codice non testato su main
  • Non condividere SSH private key
  • Non usare password per HTTPS (usa PAT o SSH)
