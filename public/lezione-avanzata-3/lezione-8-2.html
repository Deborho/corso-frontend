Creare un Nuovo Repository

Metodo 1: Inizializzare Directory Esistente

# Naviga nella directory del progetto:
cd ~/progetti/mio-sito

# Inizializza repository Git:
git init

# Output:
# Initialized empty Git repository in /Users/mario/progetti/mio-sito/.git/

Metodo 2: Creare Nuovo Progetto con Git

# Crea directory e inizializza in un comando:
git init mio-nuovo-progetto

# Entra nella directory:
cd mio-nuovo-progetto

# Output simile:
# Initialized empty Git repository in /Users/mario/mio-nuovo-progetto/.git/

Metodo 3: Inizializzare con Branch Specifico

# Inizializza con branch "main" invece di "master":
git init -b main mio-progetto

# Oppure (Git 2.28+):
git init --initial-branch=main mio-progetto

Stati dei File in Git

I file in un repository Git possono essere in uno di questi stati:

1. Untracked (Non tracciato)
   - File nuovo, Git non lo conosce ancora
   - Non Ã¨ mai stato aggiunto al repository
   - Appare come "Untracked files" in git status

2. Unmodified (Non modificato)
   - File tracciato ma non modificato dall'ultimo commit
   - Git lo ha nel database e corrisponde all'ultima versione
   - Non appare in git status (tutto ok)

3. Modified (Modificato)
   - File tracciato e modificato rispetto all'ultimo commit
   - Modifiche non ancora in staging
   - Appare come "Changes not staged for commit"

4. Staged (In staging)
   - File pronto per il prossimo commit
   - Aggiunto alla staging area con git add
   - Appare come "Changes to be committed"

Comandi Utili per Repository

# Verifica se directory Ã¨ un repository Git:
git rev-parse --is-inside-work-tree
# Output: true (se sei in un repo)

# Mostra path alla root del repository:
git rev-parse --show-toplevel
# Output: /Users/mario/progetti/mio-sito

# Mostra path alla directory .git:
git rev-parse --git-dir
# Output: /Users/mario/progetti/mio-sito/.git

# Mostra branch corrente:
git branch --show-current
# Output: main

# Verifica configurazione repository:
git config --local --list

# Rimuovi repository Git (ATTENZIONE!):
rm -rf .git  # Linux/macOS
# OPPURE
rmdir /s .git  # Windows
# Questo cancella tutta la storia Git ma mantiene i file!

âœ… Best Practices - Repository

âœ… DO - Fare:

  • Inizializza Git all'inizio del progetto, non alla fine
  • Crea un repository per ogni progetto separato
  • Usa "main" come nome del branch principale
  • Verifica con git status prima di operazioni importanti
  • Aggiungi subito .gitignore per file da non tracciare
  • Crea README.md come primo file del progetto

âŒ DON'T - Evitare:

  • Non inizializzare Git dentro un altro repository Git (no nested repos)
  • Non modificare manualmente file in .git/
  • Non committare .git/ in un altro repository
  • Non cancellare .git/ directory per "ricominciare" (usa git reset invece)
  • Non creare repository nella home directory (~)
  • Non inizializzare repository in directory di sistema (/, C:\)
